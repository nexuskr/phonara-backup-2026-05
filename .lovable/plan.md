# 프리런칭 플랜 — "0명 → 첫 100명 진성 베타"

## 냉정한 현실 인식

지금 상태:
- ✅ **기술**: CEX급 execution kernel + Oracle Fortress + Phase 3.5 shadow → **과잉 스펙**
- ❌ **유저**: 0명
- ❌ **검증**: 실제로 누가 돈을 넣고 베팅하는지 0 데이터

→ 지금부터 추가 기능은 **금지**. 14일간은 "사람을 데려오는 일"만 한다.
Phase 3.5 shadow도 어차피 14일 자동 게이트라 백그라운드에서 돌아간다 — 너는 손 뗀다.
PR-1 Heatmap도 **유저 50명 + 실거래 데이터 쌓이기 전엔 만들지 마라.** 보여줄 클러스터가 없다.

## 4주 런칭 로드맵

### Week 1 — "런칭 가능 상태" 확정 (기술 0%, 운영 100%)

**1. Closed Beta 모드 ON**
- `beta_invites` 시스템 이미 있음 → 신규 가입을 **초대코드 필수**로 강제
- 이유: 0명 → 갑자기 공개 = SEO/평판/스팸 리스크. **희소성으로 시작**
- 100개 코드 발급, 본인이 직접 분배

**2. 런칭 체크리스트 (이미 다 있는지 확인만)**
- [ ] `/trust` 페이지 — 실시간 출금 24h 통계 정상 노출되는지
- [ ] `/legal/*` 약관 3종 (이용약관/개인정보/리스크고지) 최신 날짜
- [ ] Practice Mode 진입 동선 (신규유저 3분 안에 첫 베팅 체험)
- [ ] 출금 TOTP 강제 정상 동작
- [ ] Empire 7+ Baron 승급 알림 정상 트리거
- [ ] 모바일 918px 이하 핵심 동선 (가입→입금→베팅→출금) 무결

**3. "첫 유저 페르소나" 정의 (글로 적기)**
- 누구? (예: 코인 트레이더 출신 + 게임 좋아함 + 한국어권)
- 어디 있나? (예: 디스코드 트레이딩 서버 / 트위터 KR 코인계)
- 왜 우리한테 와야 하나? (한 문장. "Binance보다 X")
→ 이거 안 정해지면 Week 2 트래픽 작업 전부 헛수고

### Week 2 — "첫 10명" 직접 데려오기 (Concierge)

**핵심 원칙**: 마케팅 X. **너가 직접 1:1로** 10명 데려온다. Paul Graham "Do things that don't scale".

**실행**:
- 친구/지인/이전 동료 중 코인 경험자 **20명에게 개인 DM**
  - "베타 테스트 부탁한다. 30분만"
  - 초대코드 + Practice Mode 링크
- **각 1명당 30분 직접 워치**
  - 화면 공유 받으며 가입 → 첫 베팅까지 옆에서 본다
  - 질문하지 말고 **막히는 지점만 메모**
  - 끝나면 "다시 올 것 같냐? 안 오면 왜?" 한 질문만

**산출물**: 
- 20명 DM → 10명 가입 목표
- "Friction Log" 1개 (막힌 지점 TOP 5)
- 이걸로 Week 3 수정 우선순위 결정

---

### Week 3 — Friction Top 5 수정 + 첫 50명

**Week 2 Friction Log 기반으로만** 수정. 그 외 기능 추가 금지.

**유저 확장**:
- Week 2 10명 중 **만족한 사람에게 코드 3개씩** 재배포
- 한국 코인 디스코드/오픈챗 2~3곳에 **글이 아닌 사람으로** 침투 (1주일 동안 그냥 대화만 → 마지막에 "사실 만들고 있는데 봐줄래?")
- 트위터 KR 코인계 마이크로 인플루언서 5명에게 **무료 Empire Baron 1주 + 코드** 제안

**KPI** (이거만 본다):
- D1 retention (가입 다음날 재방문)
- 첫 입금 전환율
- 첫 베팅까지 평균 시간
→ 이 3개가 50%/20%/10분 이하면 PMF 신호

---

### Week 4 — 데이터 기반 결정

**시나리오 A: KPI 좋음** (D1 50%+, 입금 20%+)
→ Public Open + PR-1 Heatmap 빌드 시작 (이때 데이터 충분)

**시나리오 B: KPI 나쁨** (D1 20% 이하)
→ **기능 추가 금지**. 페르소나/포지셔닝 재정의.
→ 50명에게 "왜 안 돌아왔냐" 직접 인터뷰

**시나리오 C: 입금 0**
→ 가장 위험한 신호. Trust 부족인지, 가치제안 부족인지 분리 진단.
→ Trust면 → /trust 페이지 강화, 가치제안이면 → 랜딩페이지 카피 전면 수정
