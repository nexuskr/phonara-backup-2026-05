# PHONARA Phase A RLS Policy

## 목적
Phase A에서 Supabase Row Level Security(RLS)를 적용하기 위한 최소 정책 기준을 정의합니다.

## 목표
- 사용자별 데이터 접근을 제한합니다.
- 관리자만 운영/검토용 데이터에 접근할 수 있도록 합니다.
- API 레이어의 권한 검증과 DB 수준의 방어선을 함께 구성합니다.

---

## 1. 전제
- auth.uid() 를 기준으로 사용자 식별을 수행합니다.
- users.role 값이 admin 인 경우 관리자 권한으로 간주합니다.

## 2. 정책 정의

### 2.1 users
- 사용자 본인은 자기 자신만 조회 가능
- 관리자만 전체 사용자 조회 가능

### 2.2 wallets
- 사용자 본인은 자기 지갑만 조회 가능
- 관리자만 전체 조회 가능

### 2.3 transactions
- 사용자 본인은 자기 거래 내역만 조회 가능
- 관리자만 전체 조회 가능

### 2.4 deposit_requests
- 사용자 본인은 자기 입금 요청만 조회 가능
- 관리자만 전체 조회 가능

### 2.5 withdrawal_requests
- 사용자 본인은 자기 출금 요청만 조회 가능
- 관리자만 전체 조회 가능

### 2.6 reward_history
- 사용자 본인은 자기 보상 이력만 조회 가능
- 관리자만 전체 조회 가능

### 2.7 mission_history
- 사용자 본인은 자기 미션 이력만 조회 가능
- 관리자만 전체 조회 가능

### 2.8 audit_logs
- 관리자만 조회 가능

---

## 3. 구현 원칙
- 데이터 변경은 API 레이어와 RPC 함수에서만 수행합니다.
- RLS는 조회 제한을 위한 기본 방어선으로 사용합니다.
- Phase A에서는 최소 범위의 정책부터 적용합니다.
